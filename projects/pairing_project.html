What is pair programming?

Technique where one individual (driver) writes the code and the other (navigator) directs and reviews the code.


Why is pair programming important?

  • To learn from each other
  • To have two sets of eyes while writing code
  • Driver can focus on completing current task and navigator can focus on being strategic
  • Each can act as an inspirational springboard for the other - reviewing code, helping to clarify concepts, etc.
  • To learn to how to work/cooperate well with others


Driver vs. Navigator?

Driver

  • Actually implements
  • Focuses on the syntax
  • Complete current task

Nagivator

  • How to implement
  • Focuses on the line that is being written
  • Be strategic
