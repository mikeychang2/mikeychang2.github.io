Enumerable:

The enumerable module provides collections (arrays, hashes, etc.) with the ability to search and sort.

The class the object comes from (array, hash) must have a #each method, which iterates through every members of the collection. The #each method iterates through the collection and doesn't return a value. Comparison enumerators (mix, max, sort) uses the comparison operator <=> to sort successive elements.

select / find_all

Returns a subset of the collection based on criteria in the code block (boolean) that's given 

[1,2,3,4,5].select { |num|  num.odd?  }   #=> [1, 3, 5]
            
(1..10).find_all { |i|  i % 2 == 0 }   #=> [2, 4, 6, 8, 10]

find

Returns the first occurence of the collection based on the criteria (similar to find_all) 

(1..100).find    { |i| i % 5 == 0 and i % 7 == 0 }   #=> 35
            
(1..10).detect   { |i| i % 5 == 0 and i % 7 == 0 }   #=> nil

map/collect

Transforms the collection based on the given code block and returns the results of operation 

1..4).map { |i| i*2 }      #=> [1, 4, 6, 8]
            
(1..4).collect { "dog"  }   #=> ["dog", "dog", "dog", "dog"]

reduce/inject

Accumulates the result of an operation and saves it for the next iteration. 

# Sum some numbers
            
(5..10).reduce(:+) #=> 45
            
# Same using a block and inject
            
(5..10).inject { |sum, n| sum + n } #=> 45

each_with_index

Iterates through not the actual element, but the index position of the collection. 

hash = Hash.new
            %w(cat dog wombat).each_with_index { |item, index|
              hash[item] = index
            }
            hash   #=> {"cat"=>0, "dog"=>1, "wombat"=>2}

group_by

Groups the collection by the results of the code block; returns a hash. The result of operation is the key and the elements that match the criteria as an array is the value. 

(1..6).group_by { |i| i%3 }   #=> {0=>[3, 6], 1=>[1, 4], 2=>[2, 5]}

sort

Sorts the collection by integer or alphabetical order. If used without a code block, it will default to its own comparison operator (<=>). Otherwise, you can pass a code block, which should return 0, -1, +1 based on the comparison of a and b. 

%w(bananas apples strawberries).sort #=> ["apples", "bananas", "strawberries"]
            [3, 4, 2, 5, 1].sort #=> [1, 2, 3, 4, 5]
            (1..5).sort { |a, b| b <=> a } #=> [5, 4, 3, 2, 1]

sort_by

Sorts the collection by evaluating a code block and returns a sorted array. 

%w{carrots kale celery}.sort_by { |word| word.length}#=> ["kale", "celery", "carrots"]
