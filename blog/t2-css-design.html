What are the Differences between Static, Absolute, Relative, and Fixed Positioning

Breaking Down CSS Positions

October 19, 2014

There are 4 different types of CSS positioning properties:

  • Static
  • Absolute
  • Relative
  • Fixed

There are also 4 different positioning rules: top, bottom, left or right.

It might be helpful, first, to envision every single element on the page as a block. Rectangular pixels. Positioning then is useful in moving your object - itself, a block of pixels - to exactly where you want it to go.

You define which position, which dictates how the commands are read. Then you determine how many pixels (or em, equivalent to the current font size) to shift from the top, bottom, left or right.

Static Positioning

Static is the default positioning for any elements on a Web page. If you don’t put anything, it’s static. If you apply positioning rules (top, left, bottom, right) it will be ignored. It just is.

Absolute Positioning

Absolute positioning is like pinpointing GPS coordinates for your object, then putting on it an invisibility cloak.

You determine where you want the object by defining the positioning rules. Typically, you only define two of them; so, top OR bottom, left OR right. Similar to GPS coordinates, the positioning rules dictates where your object goes. So, top:20px moves the object 20 pixels from the top of the origin, on your webpage-grid of pixels. Left:10px pushes the object 10px from the left of that same origin.

But where exactly is the origin?

For absolute positioning, the origin is relative to the first parent element defined by a position other than static. Basically, you figure out what is the parent element of your absolutely-positioned object. Is it absolute, relative, or fixed? If so, the top, left corner is now your origin for your absolutely-positioned object. If not, you move on to the next parent element. Is that absolute, relative, or fixed (AKA not static)?

If none of the parent elements have a defined position, then the browser window becomes your origin. The top:20px and left:10px then pushes the object 20px down from the top of the browser (so, basically down)and 10px to the right of the left edge of the browser.

One thing to note it that the absolutely-positioned object is completely removed from the normal flow of the page. That is, things can overlap and stack - and get a little bit messy. As far as the other elements are concerned, your absolutely-positioned doesn’t exist. It’s wearing an invisibility cloak.

Relative Positioning

Relative positioning is like a ghost-image shifted slightly off-kilter from the actual photograph.

The position is relative. But relative to what, exactly?

To itself. To the point where it would normally sit if it were static. You simply shift it the defined property (top, bottom, left, right) from where it would exist if you decided to let it be.

It’s a slight shift; hence, the slightly-altered ghost image. But like this ghost image, other elements behave as if the shift didn’t exist. They can’t see the ghost. It, thus, reacts to the original non-ghost image.

Basically, the relatively-positioned object is taken out of the normal document flow, like the absolutely-positioned object, but unlike the absolutely-positioned object, it still affects the elements around it. They just act is if it never moved at all.

Fixed Positioning

Fixed positioning is basically absolute positioning. Except that the fixed elements are then fixed in the location. Forever. It’s independent of other elements. Other elements will scroll by, but it will continue to stay in that position on your screen.
