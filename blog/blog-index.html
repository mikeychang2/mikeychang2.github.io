Diversity in Tech

Underrepresentation of Women and Minorities Hurts the Industry

November 2, 2014

When one thinks of a programmer or techie, a stereotyped image comes to mind. It is largely male. It is predominantly white. And it is increasingly Asian. These stereotypes are not far from the truth. Even a quick glance at the demographic breakdowns of the major tech companies reflect this obvious bias - where female, Black, and Hispanic populations are largely underrepresented in the ranks of the tech workforce. And even with Asian populations — itself a vast term that encompasses a huge population of the world’s largest continent — its representation among the general tech population may be over-represented; however, its inclusion in leadership drops significantly.

Enumerable Methods: Group_by

Grouping Collections based on Criteria

November 1, 2014

Enumerable methods provide collections, such as arrays or hashes, with several ways to select, sort, or transform the elements of the collection. One less discussed, but particularly useful methods the Enumerable#group_by method, which when used on an array, iterates through the collection, passing each element to the code block, which contains the criteria by which the array will be grouped. The resulting output is a hash where the key represents each of the groups and the value is an array with all the elements of the input array that fit within that criteria. Essentially, the code runs through each element within the array; based on the requirements, Ruby places it within that group.

Think of it like when you were in elementary school and the teacher has each of the students in the classroom count off from 1 through 5. All of the one’s were in one group, all the two’s in another, and so on. The criteria was which number you were. And each of the groups of students were the resulting arrays.

Arrays vs. Hashes in Ruby

Differences & Similarities

October 24, 2014

In Ruby, we store values for later use by assigning them to variables. Like num1 = 52. Or counter = 3.

Arrays and hashes are simply another, better way to organize this data.

Both are indexed collections that store objects. The key difference is how you access them. You access both through a key - the key for arrays being an integer associated with the array's numerical index, while the key for hashes being another object.

Arrays

Arrays are simply a list of things. They can be strings, integers, floats - really, any object. We store them within square brackets, where every list item is a variable. Basically, arrays are objects that contain ordered lists of other objects.
