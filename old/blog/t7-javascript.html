Ruby vs. JavaScript

Differences in Looping Methods

November 21, 2014

After spending several weeks getting accustomed to Ruby and its syntax, it was quite the challenge moving onto JavaScript. However, it is also because we spent the last couple of weeks, learning the foundation of Ruby (and programming concepts, such as Types, Booleans, and looping), that I felt a bit more comfortable understanding the less “user-friendly” syntax of JavaScript. One such concept that differs between the two languages, which would have been much more difficult to understand without basic Ruby knowledge, was looping.

With Ruby, we have the handy #each method which allows us to iterate over collections, such as arrays and hashes. The iteration over each element produces lots of handy methods utilizing the #each method, such as #map or #reduce. The #each function makes it a lot easier for us to cycle through each element of a collection, rather than using the “for" or “while" loops.

JavaScript, however, doesn’t necessarily have this same tool for iterating through collections. Rather, we do have to rely on primarily the “for” loop. The syntax, thus, requires that we input the logic with each iterative function, such as below.

Within the loop, you first create a variable i, which serves essentially as the index of the array. This is called the incrementer, which is typically set to 0, unless for some reason you don’t want to begin iterating at the start of the collection; remember that arrays are zero-origin indexed). Then, the following is where the "for" loop comes in. For when the i is under array.length, you will run the code. This is the same logic as the #each method; however, it is more manually spelled out for you. The array.length determines how many elements are within the array and you will only loop as many times as there are elements. The last bit, i++, increments the counter by one after each iteration.

With this basic “for” loop which iterates through our array, we can build on it and make more complicated functions - in the similar way that Ruby builds on the #each function for more complicated (and useful) methods.

Another difference to keep in mind is that with JavaScript doesn’t allow users to use the “unless.” Rather, you’d have to use “for !” expression which negates the expression and in effect, produces the same results as “unless."
