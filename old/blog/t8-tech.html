Ruby Regular Expressions

Basic Pattern Matching, MatchData Objects & Lookarounds

November 29, 2014

Since beginning DBC 8 weeks (EIGHT. WEEKS. Has it gone by that fast?!), I've encountered the concept of regular expressions a few times. Every time, I tried to tackle the concept within the context of learning another new Ruby concept, it was difficult to comprehend. So, now that I have a foundational understanding of Ruby now, I decided that it was time to get at least a functional understanding of the concept before the on-site phase began.

Regular expressions, or regexps, are patterns to match the contents of a string. They test whether a string has a given pattern. They also can extract portions of that match. Ruby regular expressions are Objects because everything in Ruby is an Object. This means that you can pass methods to these objects. It also means that you can set them to variables.

Basic Pattern Matching

Essentially, you are asking, "Does this string contain this pattern?" It almost serves as a search function for a string object. You're typing in the pattern, or substring, which you place within backslash.

Positive operator =~

  • Asks if this string contains this pattern
  • This returns the character number where the match begins

Negative operator !~

  • Asks if this string does NOT contain this pattern
  • This returns true/false
 /hazelnut/ =~ "cinnamon hazelnut macchiato"
=> returns 9 because that is where the character number where the match "hazelnut" begins.
                
 /vanilla/ !~ "cinnamon hazelnut macchiato"
=> returns true because the string does not contain the substring "vanilla"

MatchData Objects

Setting my_match to a MatchData Object, you have access to a lot more functionality with Regular Expressions. You are creating an instance of the MatchData Class, which allows you to access additional methods and to create, essentially, something similar to an array to access the elements within.

First, you create the my_match object.

string = "I can help the next person in line."
my_match = /person/.match(string)
=> returns

This returns in an instance of MatchData class being created, which is an Object. You can, thus, call some methods on it.***

Calling to_s produces the string of the matched pattern

 my_match.to_s
=> "person"

Calling the pre_match method returns the substring before the match

my_match.pre_match
=> "I can help the next "

Calling the post_match method returns the substring after the match

my_match.post_match
=> " in line"

Capture Groups

You can use .* which means any character appearing any number of times along with () parantheses to separate a string into groups. This will store matches for these groups to be called later on.

For instance,

/(.*)person(.*)/

The (.*) means a substring containing any character of any character length before or after the substring pattern "person".

my_match = /(.*)person(.*)/.match(string)
my_match.captures
=> ["I can help the next ", " in line"]

You can access the MatchData object my_match in the same way as an array with one key difference. The zero index returns the entire string. So, in order to access the first element of the MatchData object, you would call my_match[1].

Lookarounds

Up until now, we've looked at regular expressions in terms of basic pattern matching. However, beyond simply matching a pattern, we can also take it one step further by defining the context for that particular match. Essentially, you can return a match when and only when it is surrounded by a certain context.

The operators:

  • Positive Lookahead: ?=
  • Negative Lookahead: ?!
  • Positive Lookbehind: ?<=
  • Negative Lookbehind: ?

Lookaheads looks for the context ahead of the match for main pattern, whereas lookbehinds match a pattern everytime it is preceded by another pattern.

Reviewing Regular Expression after a more foundational understanding of Ruby proved to be more helpful. I think that with this functional background undestanding of regexps, it will be helpful with writing Ruby programs in the future, particularly with parsing through substrings within strings.
