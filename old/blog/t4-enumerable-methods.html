Enumerable Methods: Group_by

Grouping Collections based on Criteria

November 1, 2014

Enumerable methods provide collections, such as arrays or hashes, with several ways to select, sort, or transform the elements of the collection. One less discussed, but particularly useful methods the Enumerable#group_by method, which when used on an array, iterates through the collection, passing each element to the code block, which contains the criteria by which the array will be grouped. The resulting output is a hash where the key represents each of the groups and the value is an array with all the elements of the input array that fit within that criteria. Essentially, the code runs through each element within the array; based on the requirements, Ruby places it within that group.

Think of it like when you were in elementary school and the teacher has each of the students in the classroom count off from 1 through 5. All of the one’s were in one group, all the two’s in another, and so on. The criteria was which number you were. And each of the groups of students were the resulting arrays.

Example 1:

If you have a collection of numbers and you wanted to group them by a certain criteria, say whether they were odd or even, you can use the group_by method. You iterate through each element with a code block (in this case, x % 2). Those with no remainder would be even, those with a remainder of 1 would be odd.

  • Key 0 corresponds with the value - an array that includes all the numbers that had a remainder of 1 when divided by 2 (or even numbers).
  • Key 1 corresponds with the value - an array that includes all the numbers that had a remainder of 1 when divided by 2 (or odd numbers).

Example 2:

With a group of people, you can iterate through the array based on certain criteria. With the following example, I’ve split the individuals based on the number of characters in length. Then I've grouped the individuals based on whether they had the letter "o" within their names

  • Key 5 includes people whose names have 5 characters in length.
  • Key 6 includes those whose names have 6 characters in length.
  • Key False includes those who don't have the letter "o" in their names
  • Key True includes those who do have the letter "o" in their names
